## 1. КОПИРАЙТЕР

### Что работает хорошо
- Заголовок H1 "Узнай, почему твой вес стоит — и *с чего начать*" — попадает в боль
- "Я отвечаю сама — не бот, не шаблон" — снимает главное возражение
- Кейс Марины с конкретными цифрами (−4 кг, 2 недели, отёки ушли)
- Подсказки к полям: "Нужно для точности разбора", "Необязательно — помогает"

### Слабые места и замены

| Элемент | Текущий | Новый |
|---------|---------|-------|
| Подзаголовок | "Пройди короткий тест справа — и я лично разберу твою ситуацию и скажу, что именно мешает результату. Бесплатно." | "Ответь на 12 вопросов — и я лично найду, что именно блокирует твой вес. Не шаблон. Не авторассылка. Живой разбор — бесплатно." |

| Текущий | Новый |
|---------|-------|

- "дорогая" на экране успеха — устаревшее обращение

- "Хронический стресс / высокий кортизол" в Q7 — медтермин для обычной пользовательницы
- Эмодзи в вариантах ответов — в половине вопросов есть, в половине нет (непоследовательно)

## 2. МАРКЕТОЛОГ

### Где пользователи отваливаются
1. **Q7 (Диагнозы)** — медицинские термины без объяснения зачем это нутрициологу. Главная точка дропа.
2. **Q9 (Рост и вес)** — обязательный вопрос о физических параметрах без opt-out. Болезненная тема.
3. **Q12 (Контакт)** — нет email, нет выбора мессенджера. Аудитория 35–50 лет может не быть в Telegram.
4. **Мобайл** — вся левая колонка скрывается, нет соцдоказательства.

### Что добавить для конверсии
- Персонализированные реакции после ответов ("это часто связано с кортизолом")
- Email как третий вариант в Q12
- Opt-out для Q9 (рост/вес): вариант "Не хочу указывать"
- Ссылка на соцсети на экране успеха (пока ждёшь разбора)
- На мобайле — оставить видимыми "50+ женщин" и личную подпись

### Доверие к эксперту — чего не хватает
- Нет образования и регалий (только темы, не credentials)
- Один кейс (Марина) — слабо, нужно 2–3
- На мобайле доверия нет совсем — всё скрыто
- Нет ссылки на соцсети
- Фото 110×130px — слишком маленькое для "личного разбора"

### Экран успеха — что добавить
- Подтверждение что данные получены
- Конкретный срок: "Обычно в тот же день или утром следующего"
- Персонализация: "Судя по твоим ответам, Зоя уже знает с чего начать"
- Кнопка "Подписаться на Telegram Зои"

## 3. UX/UI РАЗРАБОТЧИК

### UX проблемы
- Q9 (рост/вес) обязателен — барьер, часть дропнется. Сделать `data-req="false"` или объяснить подробнее
- Q12 — два поля контакта, неясно нужно ли заполнять оба
- Кнопка "Назад" на Q1 disabled — лучше скрыть через `visibility:hidden`
- `seq.length` меняется (Q4 пропускается) — пользователь видит "из 12" → "из 11"
- Экран успеха — нет следующего шага

### UI проблемы
- `.opt:hover` → `translateX(3px)` в g2-сетке — элементы "уезжают". Заменить на `scale(1.01)`
- Кнопка "Отправить" визуально идентична "Далее" — нужен акцент
- `success-card::before` с `position: absolute` без `position: relative` у родителя
- Цвет ошибки `.err` (`#fca5a5`) — слабый контраст на тёмном фоне
- Нет анимации скрытия экрана старта — просто `display:none`

### Accessibility
- Нет ни одного `aria-*` атрибута
- `.opt` — `<div>`, не `<button>` и не `<input type="radio">`
- `<label>` не связан с полями через `for`/`id` в Q12
- `type="text"` для числовых полей — нет цифровой клавиатуры на мобайле
- Нет управления фокусом при переходе между шагами

### Код для исправлений

```css
/* position: relative для success-card */
.success-card { position: relative; }

/* translateX → scale для g2 */
.options.g2 .opt:hover { transform: scale(1.01); }
.options.g2 .opt.sel  { transform: scale(1.01); }
```

```js
/* Скрыть кнопку Назад на первом шаге */
s.querySelectorAll('.btn-back').forEach(b => {
  b.disabled = cur === 0;
  b.style.visibility = cur === 0 ? 'hidden' : 'visible';
});
```

```html
<!-- label for в Q12 -->
<label class="field-label" for="a12">Telegram или WhatsApp</label>
<input type="text" class="text-input" id="a12" autocomplete="username" ...>

<!-- type=number для Q9, Q10 -->
<input type="number" min="100" max="220" id="a9" placeholder="165">
<input type="number" min="30"  max="200" id="a9b" placeholder="72">
<input type="number" min="16"  max="80"  id="a10" placeholder="34">
```

## 4. ВРЕДНЫЙ ПОЛЬЗОВАТЕЛЬ

### Найденные дыры

**Критичные:**
- `submit()` — заглушка, данные только в `console.log`
- Поля без `maxlength` — можно вставить мегабайты текста
- Двойной вызов `next()`: клик на single-вариант + кнопка "Далее" за 300мс → квиз перепрыгивает вопрос

**Высокие:**
- Q9 (рост/вес) валидируется как Q12 (телефон): 3 символа в "Росте" = форма проходит
- Q12 принимает любые 3 символа как валидный контакт
- `ans[3]` не обновляется при возврате назад без нажатия "Далее" → `seq` может быть неверным

**Средние:**
- Нет защиты от двойного submit (когда появится fetch — улетит несколько запросов)
- Q7 "Другое" можно выбрать пустым — валидация пройдёт

**Низкие:**
- `data-skip="4"` — мёртвый атрибут, нигде не используется в JS
- CLAUDE.md содержит устаревшие баги (id="intro-screen" и id="rpanel" — уже есть в HTML)
