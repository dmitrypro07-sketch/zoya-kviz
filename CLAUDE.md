# CLAUDE.md — Квиз Зои Байковой

## Что за проект
Квиз-лендинг для нутрициолога Зои Байковой. Пользователь проходит 12 вопросов и оставляет контакт — Зоя пишет лично с разбором в течение 24 часов.

## Стек
- Чистый HTML/CSS/JS, один файл `zoya_kviz.html`
- Никаких фреймворков, никаких сборщиков
- Шрифты: Google Fonts (Playfair Display + Nunito)
- Деплой: Vercel

## Структура файла `zoya_kviz.html`
- Строки 1–10: head, мета-теги, шрифты
- Строки 10–460: `<style>` — весь CSS (переменные, левая/правая колонка, анимации)
- Строки 460–800: HTML (левая колонка + правая с Q1–Q12 + экран успеха)
- Строки 800–1000+: `<script>` — логика квиза

## Логика квиза
- `ALL_Q = [1..12]` — полная последовательность вопросов
- `seq[]` — активная последовательность (Q4 пропускается если на Q3 "Нет")
- `ans{}` — объект с ответами, ключ = номер вопроса
- `cur` — текущий индекс в seq
- `isTransitioning` — флаг защиты от двойного next()
- `_dir` — направление перехода ('forward'/'backward'/'none')
- `submit()` — пока заглушка (console.log), нужна реальная отправка

## Типы вопросов
- `data-type="single"` — один вариант, автопереход через 300мс
- `data-type="multi"` — несколько вариантов, кнопка "Далее"
- `data-type="text"` — текстовое поле
- `data-type="dual"` — два поля (Q9: рост/вес через data-key1/key2, Q12: контакт)

## Анимации (добавлены)

- Ripple при клике, pop-анимация галочки, hover glow

## Статус submit()
Данные собираются в `ans{}`, но отправки нет — ждём токен Telegram-бота.
Когда придёт токен — вставить fetch в функцию submit() (строка ~915).

## Правила разработки
- Не трогать base64 фото Зои без необходимости (оно огромное)
- Стиль CSS — CSS-переменные в `:root`, не хардкодить цвета
- JS — без библиотек, чистый vanilla JS
- Все исправления и рекомендации — в research.md

## Команды запуска
```
Открыть zoya_kviz.html в браузере — локально
Деплой: vercel --prod (из папки проекта)
```

## Известные ограничения
- base64 фото делает файл ~700KB — медленная загрузка
- Нет бэкенда — только фронтенд, данные нигде не хранятся до подключения submit()
